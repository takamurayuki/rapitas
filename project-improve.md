# Rapitas プロジェクト評価プロンプト

## 評価概要

このプロンプトは、Rapitas（階層型タスク管理 + AI支援開発・学習プラットフォーム）を厳格に評価し、改善点を特定するためのものです。

## B. 差別化・独自価値（100点）

既存の競合アプリ（Todoist, Notion, Linear, Asana, TickTick, Obsidian Tasks）との差別化を評価します。

### B-1. 競合との比較分析（40点）

以下の競合と比較し、**優位性**を評価:

| 競合 | 主な強み | Rapitasの優位性チェック |
|------|---------|------------------------|
| Todoist | シンプルさ、自然言語入力 | より高度な機能を提供しつつ複雑すぎないか |
| Notion | 柔軟性、ドキュメント統合 | タスク特化による深い機能性 |
| Linear | 開発者向けUI、高速性 | AI統合による自動化 |
| Asana | チームコラボ、ワークフロー | 個人〜小規模チーム向け差別化 |
| TickTick | 習慣トラッキング、ポモドーロ | AI + 学習支援の統合 |

**評価項目:**
- [ ] 10点: 明確な競合優位性が3つ以上
- [ ] 7点: 競合優位性が1-2つ
- [ ] 3点: 同等レベル
- [ ] 0点: 競合より劣る

### B-2. 独自機能の価値評価（60点）

| 独自機能 | 配点 | 評価基準 |
|---------|------|---------|
| AI自動タスク分解 | 15点 | 実用的か、精度は十分か |
| Claude Code統合 | 15点 | 開発者ワークフローに真に統合されているか |
| 学習支援機能 | 10点 | 単独アプリと比較して価値があるか |
| GitHub連携 | 10点 | Issue/PR同期の完成度 |
| 承認ワークフロー | 10点 | AI安全性と実用性のバランス |

**致命的減点項目:**
- 「使われない機能」が50%以上: -20点
- 機能過多による複雑化: -15点
- 差別化機能が動作不完全: -10点

## 評価実施手順

### Step 1: 自動チェック

```bash
# コード品質チェック
npm run lint
npm run type-check

# テストカバレッジ
npm run test:coverage

# バンドルサイズ分析
npm run analyze

# 依存関係の脆弱性
npm audit
```

### Step 2: 手動レビュー

1. **機能テスト**: 各機能を実際に操作して動作確認
2. **UXテスト**: 初見ユーザー視点での操作
3. **競合比較**: Todoist/Notion/Linearを並行使用して比較
4. **AI評価**: 10個のタスクでAI分解精度を測定

### Step 3: スコア集計

```
最終スコア = Σ(カテゴリスコア × 重み)

評価ランク:
- S (90-100点): 市場投入準備完了、競争力あり
- A (80-89点): 軽微な改善で市場投入可能
- B (70-79点): 中規模の改善が必要
- C (60-69点): 大幅な改善が必要
- D (60点未満): 根本的な見直しが必要
```

## 致命的欠陥チェックリスト

以下に該当する場合、**即座に対応が必要**（スコアに関係なく）:

### 🔴 Critical（即時対応）

- [ ] データ消失の可能性があるバグ
- [ ] セキュリティ脆弱性（SQLインジェクション、XSS等）
- [ ] APIキーのハードコーディング
- [ ] 本番環境でのデバッグモード有効化

### 🟠 High（1週間以内）

- [ ] 主要機能の動作不良
- [ ] パフォーマンス問題（3秒以上の遅延）
- [ ] エラー時のユーザー通知なし
- [ ] データの不整合発生

### 🟡 Medium（1ヶ月以内）

- [ ] 非直感的なUI操作
- [ ] 不十分なエラーメッセージ
- [ ] ドキュメントの不備
- [ ] テストカバレッジ60%未満
